For this homework assignment you will be creating a simple Angular application that simulates an online chat. This amazing chat program will allow two people sitting at the same computer to talk to one another as if they were sitting right next to each other!

The web page will consist of two terminals, each containing a text box that the user can type messages into. When the user enters a message, it will be added to a display at the bottom of the page. This display will show all the messages that have been entered, from most recent to least recent, each labeled with the name of the user who submitted the message.

You should create and run this assignment using the Angular CLI as shown in class. The focus of this program is on creating Angular components and sharing information between them. You must create the following components:

  1. The Display

    This component will show all the messages that have been typed from most recent to least recent. Each message will be labeled with the user who submitted it. The display component should have an input property for the array of messages. It should not alter the messages.

  2. The Terminal

    This component will allow one user to enter messages. It consists of a text input labeled with the name of the user. The terminal component should have an input property for the name of the user, and an output event that is triggered when the user enters a message.

    Because Angular is an opinionated framework, I recommend that you do not use a form in the template of this component. (Angular has opinions on how forms should be used in an application.) Instead, you may create a button the user can click to submit a message, or you may register a handler for the special angular event keydown.enter to trigger an event every time the user presses the Enter key. 

    <input type="text" (keydown.enter)="submitMessage()"/>

    You should not submit an empty message; if the text box is empty when the user clicks the button and / or presses the Enter key then do nothing. Any time a message is submitted you should clear out the input box so that it is ready for the next message.

  3. The application component

    This is your root component. It should contain two terminal components (one for user "One" and one for user "Two") and a display.

    This component should be in charge of managing the list of messages. It should create the initial list and give it to the display component. Then, any time one of the terminals fires its event indicating a new message has been submitted it should add it to the list. (You may want to use the array's unshift method.)

Use CSS to make your components look nice. You should have at least one CSS rule for each component.
